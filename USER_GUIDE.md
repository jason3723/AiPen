## 一、认识 AiPen

### 1.1 什么是 AiPen

AiPen 是一款面向**公文、报告、讲话稿等中文公务写作场景**的桌面端 AI 辅助应用。它不仅仅是 ChatGPT 的套壳——它把"提问采集 → 提纲生成 → 正文撰写 → 审查润色"这一完整写作流水线，与"富文本编辑、素材管理、知识库、版本对比"等专业写作工具**整合在同一个窗口**中，让你无需在多个软件之间来回切换。

### 1.2 核心定位

- **不是**简单的 AI 对话工具
- **是**一套完整的 **"AI 协作式写作工作流"**——围绕"用户提需求 → AI 出草稿 → 人工审改 → AI 再润色"循环优化
- 专为中文公文写作场景深度定制，覆盖 10 种常见文种

### 1.3 适用人群

- 机关、企事业单位的**公文写作者**
- 需要频繁撰写**讲话稿、汇报材料、调研报告、通知公告、规章制度**的同志
- 任何想要用 AI 提升**长文写作效率与质量**的用户

### 1.4 与普通 AI 工具的本质区别

| 维度 | 普通 AI 工具 | AiPen |
|------|-------------|-------|
| 工作方式 | 一问一答、零散对话 | 五阶段流水线，结构化产出 |
| 文种支持 | 通用提示词，需用户自行设计 | 10 种内置菜谱，每种有专属信息采集问题 |
| 质量控制 | 无系统化审查 | 内置逻辑/语法/规范 triple 审查 |
| 版本管理 | 无 | 完整版本历史、Diff 对比、回退 |
| 排版导出 | 需手动排版 | 一键按公文规范导出 Word |
| 知识沉淀 | 无 | 知识库 + 素材库，AI 自动调用 |

## 二、上手准备：API 密钥与账户充值

AiPen 使用 **DeepSeek 大模型**作为 AI 引擎。你需要一个 DeepSeek API 密钥才能使用所有 AI 功能。

### 2.1 注册 DeepSeek 账号

1. 打开浏览器，访问 DeepSeek 开放平台：[https://platform.deepseek.com](https://platform.deepseek.com)
2. 点击页面上的「注册」按钮
3. 支持**手机号注册**或**邮箱注册**
4. 填写验证码，设置密码，完成注册
5. 登录后进入控制台

### 2.2 完成实名认证

> **注意**：实名认证是必选项，未认证无法创建 API Key。

1. 在控制台左侧找到「设置」→「实名认证」
2. 按页面提示提交身份信息（通常需要姓名、身份证号）
3. 提交后等待审核（一般几分钟到几小时）
4. 审核通过后，账户状态变为"已认证"

### 2.3 账户充值

> **注意**：新账号余额为 0，必须先充值才能调用 API。DeepSeek 采用**预付费**模式。

1. 在控制台左侧找到「设置」→「充值」
2. 选择充值金额：
   - **首次建议**：充值 10~50 元测试使用。DeepSeek 价格非常低廉，几十元可以生成大量文本
   - **参考用量**：V4 Flash 模型价格约 0.3 元 / 百万 tokens。一份 5000 字的公文，从采集到润色全流程走完约消耗几分钱
3. 选择支付方式（微信/支付宝）
4. 完成支付后，余额即时到账

### 2.4 创建 API Key

1. 在控制台左侧点击「API Keys」
2. 点击「创建 API Key」按钮
3. 输入一个名称（如"aipen"），便于识别
4. 点击确认创建
5. **立即复制生成的密钥**，格式为 `sk-xxxxxxxxxxxxxxxxxxxxxxxx`
6. **⚠ 重要**：密钥仅在创建时显示一次，请务必复制并妥善保存！如果丢失，只能删除重建

### 2.5 在 AiPen 中配置 API

1. 启动 AiPen
2. 点击右侧栏的「**设置**」标签
3. 在「DeepSeek API 设置」区域：
   - **API 密钥**：粘贴上一步复制的 `sk-...` 密钥
   - **模型**：选择合适的模型
   - **API 地址**：默认已填写 `https://api.deepseek.com`，无需修改
4. 点击「**保存设置**」
5. 点击「**测试连接**」确认配置正确——成功会显示"连接成功"

### 2.6 两个模型的区别

| 项目 | V4 Flash（快速模式） | V4 Pro（专家模式） |
|------|---------------------|--------------------|
| 速度 | 快 | 相对较慢 |
| 质量 | 良好，适合日常任务 | 深度品质，适合复杂写作 |
| 适用场景 | 问答采集、提纲生成、一般正文 | 重要文稿、正式定稿、复杂分析 |
| 推荐 | 默认使用 | 关键文档可切换 |

> **技巧**：日常写作用 Flash 模式更加高效，遇到关键文档（如正式呈报、公开发表）时切换 Pro 模式。

### 2.7 思考模式与思考强度

- **思考模式**：开启后，AI 会在生成内容前先进行深度推理（可看到推理过程），输出质量更高但响应稍慢
- **思考强度**：
  - **high（高）**：适合大多数场景，兼顾质量与速度
  - **max（最强）**：适合复杂写作任务，推理更充分但耗时更长
- **注意**：思考强度仅在开启思考模式后生效；关闭思考模式时此选项不可用

> **技巧**：
> - 问答采集阶段 → 建议关闭思考模式，快速推进
> - 正文撰写阶段 → 建议开启思考模式（high），提高质量
> - 关键段落→ 可临时切换 max，写完再切回 high

### 2.8 查询 API 余额

在设置面板中，配置好 API 密钥后，点击余额旁边的刷新按钮（↻），即可查询当前账户余额。建议定期关注余额，避免写作到一半因欠费中断。

### 3.1 顶栏（应用级操作）

顶栏负责应用级别的核心操作：

| 元素 | 详细说明 |
|------|----------|
| **AiPen 图标** | 应用图标，点击无特殊操作 |
| **文档标题** | 显示当前编辑的文档名称。**单击**进入重命名模式，输入新名称后回车保存；**右键**可进行复制、剪切等操作 |
| **⭐ 评分** | 对当前文档进行五维度综合评分（0~100 分）。评完分数颜色区分：绿色（80+ 优秀）、黄色（60~79 良好）、红色（<60 需改进）。点击分数可展开查看各维度详细评语 |
| **导出 Word** | 按公文排版规范将当前文档导出为 `.docx` 文件。弹出保存对话框选择路径即可 |
| **⚙ 排版** | 打开排版设置弹窗，调整各级标题字体字号、正文字号、行距、页边距、标题风格等（详见第十三章） |
| **提交版本** | 将当前编辑器内容保存为一个**不可变的快照版本**，可附加备注说明。这是写作里程碑管理的关键操作 |
| **_ □ × ** | 窗口控制：最小化 / 最大化 / 关闭。关闭窗口后应用在后台运行（Tauri 托盘），数据自动保存 |

### 3.2 左栏（资源管理）

左栏包含三个标签页，管理所有写作资源：

#### 3.2.1 文档标签

- **文件夹树**：支持创建文件夹来组织文档。右键文件夹可重命名、删除
- **筛选**：点击文件夹名筛选该文件夹下的文档，点击「全部」显示所有文档
- **新建文档**：点击顶部 **+ 新建** 按钮创建新文档
- **文档列表**：按更新时间倒序显示，单击切换文档，右键可重命名、删除、移动到文件夹

#### 3.2.2 素材标签

- **标签导航**：左侧栏展示所有素材标签，点击标签查看该标签下的素材
- **新建标签**：点击 + 按钮创建新标签
- **标签右键**：重命名、删除标签
- **查看标签文档**：点击标签旁的文档图标，将该标签下所有素材合并为只读文档展示
- **素材列表**：按更新时间倒序展示，单击进入编辑模式
- 详细使用方法见[第六章：素材库管理](#六素材库管理)

#### 3.2.3 浏览器标签

- **内嵌 WebView**：在应用内浏览网页，无需切换窗口
- **地址栏**：输入网址回车即可打开
- **书签栏**：收藏常用网址，点击即可跳转
- **网页剪藏**：浏览网页时选中文字，右键 →「存入素材库」
- 详细使用方法见[第十六章：内嵌浏览器与网页剪藏](#十六内嵌浏览器与网页剪藏)

### 3.3 中央编辑器

基于 **TipTap（ProseMirror）** 专业富文本编辑器，是实现专业写作的核心工作区。工具栏按功能分组：

#### 3.3.1 导航区

| 按钮 | 功能 |
|------|------|
| **☰ 大纲** | 点击展开/收起文档大纲侧边栏。大纲自动从文档的 H1~H4 标题生成，点击大纲中的条目可快速跳转到对应位置。适合长文导航和结构审查 |

#### 3.3.2 编辑历史

| 按钮 | 快捷键 | 功能 |
|------|--------|------|
| **↩ 撤销** | Ctrl+Z | 撤销上一步操作 |
| **↪ 重做** | Ctrl+Y | 恢复被撤销的操作 |

#### 3.3.3 基础文字格式

| 按钮 | 快捷键 | 功能 |
|------|--------|------|
| **B 加粗** | Ctrl+B | 文字加粗 |
| **I 斜体** | Ctrl+I | 文字斜体 |
| **U 下划线** | Ctrl+U | 文字下划线 |
| **S 删除线** | — | 文字加删除线（~~划掉~~） |

#### 3.3.4 高级文字样式

| 按钮 | 功能 | 使用场景 |
|------|------|----------|
| **上标** | 文字上标显示（X²） | 数学公式、注释编号 |
| **下标** | 文字下标显示（H₂O） | 化学式、脚注 |
| **文字颜色** | 更改文字颜色 | 标注重点、批注标记 |
| **高亮** | 文字背景高亮（荧光笔效果） | 标记关键句、待修改内容 |
| **清除格式** | 移除选中文字的所有格式 | 粘贴带格式文本后的清理 |

#### 3.3.5 段落样式

| 按钮 | 功能 |
|------|------|
| **标题下拉** | 设置当前段落级别：正文（默认）、标题 1（H1）、标题 2（H2）、标题 3（H3）、标题 4（H4）。H1~H4 会自动出现在大纲导航中 |

> **技巧**：写长文时，养成先设定标题层级的习惯。H1 用于一级标题（如"一、总体情况"），H2 用于二级标题（如"（一）生产经营"），H3 用于三级标题。这样做的好处是：
> 1. 大纲导航一目了然
> 2. 导出 Word 时自动应用对应的标题格式
> 3. AI 写作流水线生成的提纲可以直接设好层级

#### 3.3.6 段落排列

| 按钮 | 功能 |
|------|------|
| **左对齐** | 段落左对齐（默认） |
| **居中对齐** | 段落居中对齐（常用于标题） |
| **右对齐** | 段落右对齐（常用于署名、日期） |
| **两端对齐** | 段落两端对齐（公文正文标准格式） |
| **↪ 缩进** | 增加段落缩进 |

#### 3.3.7 列表

| 按钮 | 功能 |
|------|------|
| **无序列表** | 创建项目符号列表（•） |
| **有序列表** | 创建数字编号列表（1. 2. 3.） |

#### 3.3.8 插入元素

| 按钮 | 功能 | 注意事项 |
|------|------|----------|
| **分割线** | 插入水平分割线 | 用于分隔章节 |
| **表格** | 插入表格，弹出行列数选择器 | 插入后点击表格会出现表格工具栏，支持增加/删除行列、合并单元格。Ctrl+点击可选中单元格 |
| **图片** | 从本地选择图片插入文档 | 支持常见格式（png/jpg/gif/webp）。导出 Word 时图片会一并嵌入 |

#### 3.3.9 文档工具

| 按钮 | 快捷键 | 功能 |
|------|--------|------|
| **🔍 查找** | Ctrl+F | 打开查找/替换面板，支持正则搜索、大小写敏感切换 |
| **🖨 打印** | — | 调用系统打印对话框 |

#### 3.3.10 字体与字号

| 控件 | 功能 | 说明 |
|------|------|------|
| **字体下拉** | 选择当前文字的字体 | 提供等线、微软雅黑、宋体、黑体、楷体、仿宋等 10 种中文字体 |
| **字号下拉** | 选择当前文字的字号 | 采用中文号数制（小初 → 五号），与公文排版标准对齐 |

> **注意**：字体和字号的设置会影响**文档实际排版**和**导出 Word 的效果**，不像页面缩放只影响显示。

#### 3.3.11 页面缩放

| 控件 | 操作方式 | 说明 |
|------|----------|------|
| **[−] NN% [+]** | 点击按钮 / Ctrl+滚轮 | 缩放范围 50%~200%，步进 5% |

> **注意**：页面缩放是**等比放大/缩小显示**，不影响文档实际字号和导出效果。如需调整正文大小，请使用字号下拉。

### 3.4 右栏（AI 协作面板）

右栏包含 8 个标签页，按典型写作工作流顺序排列：

| 标签 | 图标 | 核心功能 |
|------|------|----------|
| **版本** | 📋 | 查看历史版本列表、提交新版本、选择版本进行对比、查看历史版本内容 |
| **Diff** | ↔ | 逐行对比任意两个版本的差异，新增/删除/修改三种状态分别用不同颜色高亮 |
| **AI分析** | 🔍 | 对选中的两个版本之间的修订进行 AI 多维度评析（政治站位、逻辑结构、见解深度、语言表达等） |
| **AI对话** | 💬 | 与 AI 自由对话，支持多轮上下文、引用全文/选区、引入知识库/素材库、附件上传 |
| **技能** | ⚡ | 调用预设或自定义 AI 技能对文稿进行专项处理，支持技能组合管道 |
| **写作** | ✍️ | 启动"菜谱式"智能写作流水线，选择文种后按五阶段生成文稿（核心功能） |
| **知识库** | 📚 | 管理本地知识文档（txt/md/docx/pdf），作为 AI 写作的参考上下文 |
| **设置** | ⚙ | AI 模型配置、数据导入导出、软件更新、API 帮助 |

### 3.5 底部状态栏

| 元素 | 说明 |
|------|------|
| **保存状态** | 实时显示草稿保存流转：`检测到变更…(黄)` → `正在保存…(蓝)` → `草稿已自动保存(时间)(绿)` → 异常时显示红色错误信息 |
| **字数统计** | 显示当前文档的总字符数（含标点） |
| **字体 / 字号** | 快捷设置当前选区的字体和字号，与编辑器工具栏中的控件功能一致 |
| **页面缩放** | 快捷缩放按钮，与编辑器工具栏中的缩放控件功能一致 |
| **🌙/☀️ 主题** | 点击切换浅色/深色主题，长按或右键可切换"跟随系统"模式 |

## 四、文档管理

### 4.1 创建新文档

1. 点击左栏「文档」标签
2. 点击顶部的 **+ 新建** 按钮
3. 输入文档标题（如"2025年度述职报告"），回车确认
4. 新文档会自动创建并立即打开

### 4.2 文档文件夹

为了更好地组织文档，你可以创建文件夹来分类管理：

**创建文件夹**：
- 文档列表顶部，点击文件夹图标旁的 + 按钮
- 输入文件夹名称（如"2025年度"、"述职材料"），回车确认

**移动文档到文件夹**：
- 右键点击文档列表中的文档 → 选择「移动到文件夹」→ 选择目标文件夹
- 也可以拖拽文档到目标文件夹

**移出文件夹**：
- 右键点击文件夹内的文档 → 选择「移出文件夹」，文档回到未分类

**文件夹操作**：
- **重命名**：右键文件夹 → 重命名
- **删除**：右键文件夹 → 删除（文档不会删除，只是移出文件夹）
- **筛选**：点击文件夹名只显示该文件夹内的文档；点击「全部」显示所有文档

### 4.3 切换文档

- 点击文档列表中的任意文档即可切换
- 切换后编辑器自动加载该文档的最新草稿
- 如果之前有未完成的草稿，会自动恢复

### 4.4 重命名文档

有两种方式：
- 点击顶栏的文档标题 → 进入编辑模式 → 输入新名称 → 回车保存
- 右键文档列表中的文档 → 选择「重命名」

### 4.5 删除文档

- 右键文档列表中的文档 → 选择「删除」
- 删除操作**不可撤销**，建议删除前先导出备份

### 4.6 自动保存与草稿恢复

- 所有修改**自动保存**，无需手动 Ctrl+S
- 防抖延迟 1 秒：你停止输入 1 秒后自动写入磁盘
- 状态栏实时显示保存状态流转
- 草稿与正式版本**分开存储**：日常编辑存在草稿中，点击「提交版本」后才生成不可变的历史版本
- 关闭应用或切换到其他文档时，当前草稿已自动保存，不会丢失

## 六、素材库管理

素材库是 AiPen 的"弹药库"——把日常收集的好词好句、政策文件原文、经典案例等统一管理，供 AI 写作时自动调用。

### 6.1 理解素材库的结构

素材库采用**标签制**组织：
- 每个素材可以标记**多个标签**（如"领导讲话"+"安全生产"）
- 标签可以自由创建、重命名、删除
- 一个标签下可以包含多个素材
- 标签文档视图：将一个标签下所有素材合并为只读文档查看

### 6.2 创建标签

1. 切换到左栏「素材」标签
2. 在标签列表中点击 **+** 按钮
3. 输入标签名称（如"领导讲话"、"政策原文"、"经典案例"、"习语引用"）
4. 回车确认

**标签命名建议**：
- 按内容类别：领导讲话、政策文件、案例数据、金句名言
- 按使用场景：述职素材、总结素材、讲话素材
- 按来源：上级文件、新闻摘录、内部资料

### 6.3 存入素材的三种方式

#### 方式一：编辑器右键存入

1. 在编辑器中**选中需要收藏的文字**
2. 右键 →「存入素材库」
3. 在弹出的对话框中：
   - AI 会自动分析内容并**推荐合适的标签**（高亮显示）
   - 你可以选择推荐标签，也可以手动选择或创建新标签
   - 素材标题会自动从选中文字的前几个字生成，可以修改
4. 点击确认存入

> **技巧**：写作过程中发现写到某处时可以用到一个之前看过的表达，立刻选中→右键→存入素材库。不需要等到写完再做。

#### 方式二：浏览器剪藏

1. 切换到左栏「浏览器」标签
2. 在内嵌浏览器中打开目标网页
3. **选中需要的文字**
4. 右键 →「存入素材库」
5. 系统会自动记录**来源网址**和**来源标题**，方便后续追溯
6. 选择标签，确认存入

#### 方式三：手动新建

1. 在素材标签页中点击「新建素材」
2. 输入标题
3. 粘贴或手动输入素材内容
4. 选择标签
5. 保存

### 6.4 浏览与编辑素材

- **素材列表**：按更新时间倒序展示，显示标题、标签、更新时间
- **筛选**：点击左侧标签名，只显示该标签下的素材
- **编辑**：单击素材条目进入编辑模式，修改后自动保存
- **删除**：素材条目右侧有删除按钮

### 6.5 标签文档视图

这是素材库的特色功能——将某个标签下的所有素材**合并为一份只读文档**：

1. 在标签列表中，找到目标标签
2. 点击标签旁边的文档图标
3. 该标签下所有素材按顺序合并展示为只读文档
4. 适合**通读复习**所有相关素材，或**导出参考**

### 6.6 素材如何被 AI 使用

在写作流水线的阶段一（问答采集）中，你可以勾选要引用的素材标签：
- AI 会根据你选择的标签，自动检索相关素材
- 在正文撰写时，AI 会将匹配的素材作为上下文参考
- 素材内容**不会原样照抄**，AI 会消化理解后用自己的语言表达

> **技巧**：日常养成"看到好内容就存"的习惯。每天花几分钟把看到的政策文件、好句子存入素材库并打上标签。一个月后，你就会拥有一个个人专属的高质量写作"弹药库"。

## 七、智能写作流水线（核心功能）

> 这是 AiPen 区别于普通 AI 工具的最大特色，也是你提升写作效率的核心武器。

### 7.1 什么是"菜谱式"写作

传统 AI 写作方式：你给 AI 一个提示词 → AI 生成一段文字 → 你反复修改提示词 → AI 反复生成。整个过程零散、随机、质量不可控。

AiPen 的"菜谱式"写作：将"写一篇公文"拆解为**五个标准化阶段**：

```
阶段一        阶段二        阶段三        阶段四        阶段五
问答采集  →  提纲生成  →  正文撰写  →  质量审查  →  润色定稿
Interview    Outline      Generate     Review       Polish
```

每个阶段有明确的输入输出，AI 在每步都有特定的任务和角色。你不用自己设计提示词，选择菜谱类型即可。

### 7.2 启动流水线

1. 点击右栏「**写作**」标签
2. 浏览 10 种内置菜谱（见 7.3），找到适合你当前写作任务的类型
3. 点击菜谱卡片，进入流水线工作台
4. 此时会弹出一个**对话式界面**，AI 开始问第一个问题

### 7.3 十种内置菜谱

| 菜谱 | 图标 | 适用场景 | 关联知识库 |
|------|------|----------|------------|
| **述职报告** | 🏛️ | 个人年度/任期/班子/党建/安全等各类述职报告 | kb_003, kb_006, kb_007 |
| **领导讲话** | 📋 | 动员部署、主旨主题、会议总结、调研座谈等讲话稿 | kb_001, kb_002, kb_003, kb_004 |
| **工作总结** | 📈 | 年度/半年/季度/专项工作总结 | kb_003, kb_004, kb_005 |
| **研讨材料** | 💡 | 理论学习中心组、民主生活会、专题研讨发言材料 | kb_006, kb_001 |
| **通讯简报** | 📰 | 工作动态、会议报道、经验交流等简报 | kb_001, kb_008, kb_013 |
| **课题研究** | 🔬 | 调研报告、课题研究、可行性研究、对标分析 | kb_003, kb_004, kb_005 |
| **工作通知** | 📢 | 工作部署、会议通知、任免通知等 | kb_003 |
| **规章制度** | 📜 | 管理办法、实施细则、操作规程、应急预案 | kb_003 |
| **经验材料** | ⭐ | 工作汇报、经验交流、典型发言、成果展示 | kb_007, kb_008, kb_009 |
| **工作汇报** | 📊 | 向上级汇报综合工作、专项任务、项目进展 | kb_003, kb_004, kb_005 |

> **提示**：每种菜谱的问题列表都是**为该文种专门设计**的，覆盖了该文种最容易遗漏的关键信息。比如述职报告会问述职人职务、述职周期、KPI 完成情况、HSE 责任、党建情况等。

### 7.4 阶段一：问答采集（Interview）—— 详细操作指南

这是整个流水线的**唯一差异化入口**。不同文种之间的差异全部收敛在这个阶段。

#### 7.4.1 问答流程

1. AI 逐题提问：菜谱预设了 10~16 个针对性问题，AI 会**一次一个问题**地来问你
2. 你逐一作答，按回车或点击「发送」提交
3. 对于**不必回答**的问题（标记为"可选"），可以点击「跳过」，AI 会自行判断补充
4. 对于**选择题型**的问题（有选项按钮），可以直接点击选项快捷回答

#### 7.4.2 问题类型

- **必答题**：标记为红色星号，通常包括文种类型、主体身份、时间范围等基础信息
- **选答题**：可以选择跳过，AI 会基于已有信息和知识库补充
- **选择题**：提供预设选项按钮，点击即可快速回答（如讲话时长、领导风格等）
- **自由文本题**：输入任意文字作答

#### 7.4.3 常用提示词（快捷输入）

每道问题旁有一个「提示词」按钮，点击可以：
- 选择已保存的**常用答案模板**
- 新建提示词：输入标签和内容，保存后下次可快速插入
- 编辑/删除已有提示词

> **技巧**：如果你经常写同一类型的述职报告，可以把常用的"分管业务描述"、"HSE 情况描述"等保存为提示词。下次写述职报告时，一键插入，不用重复打字。

#### 7.4.4 补充说明

所有问题回答完毕后，会进入**补充说明环节**：
- 你可以在这里补充之前漏掉的任何信息
- 也可以直接跳过

#### 7.4.5 信息摘要确认

补充说明提交后，会弹出一个**摘要窗口**，展示：
- 你之前**所有问题的回答**，可以在此回顾并**直接编辑**修改任何一条
- 当前菜谱**关联的知识库**（可增减勾选）
- 要引用的**素材库标签**（可增减勾选）
- 一个**自由文本框**用于补充其他信息

> **重要**：这是一个关键的"检查点"。建议在这里花 1~2 分钟仔细过一遍所有回答，确保没有遗漏或错误的关键信息。信息准确，后续提纲和正文质量才会高。

### 7.5 阶段二：提纲生成（Outline）—— 详细操作指南

#### 7.5.1 输入文章结构意图

进入提纲阶段后，会先让你描述期望的**文章结构**：
- 系统会给出一个参考结构（如"形势分析→回顾成绩→部署任务→提出要求→号召动员"）
- 你可以直接确认，也可以输入自定义结构
- 这一步决定了提纲的整体框架

#### 7.5.2 选择标题风格

AiPen 内置了 **14 种标题风格**，分为 6 大类：

| 风格类别 | 包含风格 | 说明 |
|----------|----------|------|
| **结构强化型** | 并列排比式、对仗工整式、统一后缀式 | 通过句式结构强化标题节奏 |
| **逻辑关系型** | 破立转换式、辩证统一式、层层递进式 | 通过逻辑关系组织标题内容 |
| **意象赋能型** | 生动比喻式、引用典故式、拟人动感式 | 用比喻、典故、拟人增强感染力 |
| **词汇创新型** | 化用热词式、概念组合式 | 引入时代热词和新概念 |
| **价值引领型** | 点明宗旨式、鼓舞动员式 | 突出理念和鼓舞性 |
| **数字统领型** | 数字概括式 | 用数字概括核心要点 |

此外还有**随机类型**（AI 自由发挥）和**不指定类型**两种选项。

二级标题（H2）和三级标题（H3）可以**分别选择不同风格**。

> **技巧**：
> - 述职报告、工作总结 → 推荐"并列排比式"或"对仗工整式"
> - 领导讲话、动员部署 → 推荐"鼓舞动员式"或"层层递进式"
> - 研讨材料、理论文章 → 推荐"辩证统一式"或"引用典故式"
> - 经验材料、典型发言 → 推荐"生动比喻式"或"数字概括式"

#### 7.5.3 AI 思考过程

点击「生成提纲」后，AI 首先输出一段**思考过程**（如果开启了思考模式），展示它如何组织结构和选择标题，然后逐条生成提纲条目。

#### 7.5.4 编辑提纲条目

提纲生成后，你可以：
- **内联编辑**：点击任意提纲条目直接修改文字
- **增删条目**：添加新条目、删除不需要的条目
- **调整层级**：将二级标题降为三级，或将三级升级为二级
- **追加用户要求**：在每条提纲上追加补充要求（如"此处需要有具体数据"），AI 在写正文时会参考
- **整体反馈**：如果不满意，可以输入反馈意见让 AI 重新生成

#### 7.5.5 导入/导出提纲

- **导出**：点击导出按钮，将提纲保存为 JSON 文件
- **导入**：点击导入按钮，加载之前导出的提纲
- 提纲文件可以在不同菜谱/不同文档间共享

> **技巧**：提纲是可以反复迭代的。如果第一次生成的提纲不太满意，不要直接接受——先尝试修改风格，或者给 AI 反馈让它重新生成。通常迭代 1~2 次就能得到满意的结果。

### 7.6 阶段三：正文撰写（Generate）—— 详细操作指南

#### 7.6.1 生成流程

提纲确认后，点击「开始生成」，AI 会按照提纲**逐节撰写**正文：
- **流式输出**：内容实时推送到编辑器，你可以看到 AI "正在写"的过程
- **逐节推进**：AI 按提纲顺序一节一节地写，每节写完自动进入下一节
- **进度指示**：界面上显示当前进度（如"正在撰写第 3/8 节"）

#### 7.6.2 生成控制

| 操作 | 说明 |
|------|------|
| **暂停** | 点击暂停按钮，AI 停止当前生成。可以查看已生成的内容 |
| **继续** | 点击继续按钮，从暂停处接着生成 |
| **重新生成某节** | 如果某一节写得不满意，可以单独让 AI 重新生成那一节 |
| **修改提纲后重新生成** | 如果对已生成的部分不满意，可以回到提纲阶段修改后重新生成 |

#### 7.6.3 不同文种的生成策略

每种菜谱在正文撰写阶段使用了不同的 temperature（温度）参数：

| temperature 范围 | 效果 | 适用文种 |
|-----------------|------|----------|
| 0.3 | 最保守，贴近模板 | 工作通知、规章制度 |
| 0.4~0.5 | 较保守，保持规范 | 课题研究、工作总结、述职报告、工作汇报 |
| 0.6 | 中等创造性 | 研讨材料、通讯简报 |
| 0.7~0.8 | 较高创造性 | 领导讲话、经验材料 |

> **技巧**：正式制度类文档（通知、规章）建议接受 AI 生成的规范表述；讲话类、经验类则可以在生成后进行更多人工修改润色。

### 7.7 阶段四：质量审查（Review）—— 详细操作指南

正文生成完成后，AI 进入质量审查阶段。这是 AiPen 最独特的质量控制环节——**三个专项技能依次审查**：

#### 7.7.1 审查流程

三个审查技能**按顺序依次执行**，每个技能审查完后会自动修复发现的问题：

```
逻辑审查 → 语法审查 → 规范审查
(skill_logic)  (skill_grammar)  (skill_official)
```

#### 7.7.2 审查维度

| 维度 | 审查内容 | 示例 |
|------|----------|------|
| **逻辑审查** | 论点是否统一、论证是否充分、前后是否矛盾、层级关系是否正确 | "第一段说'产量增长 10%'，第三段又说'产量基本持平'——矛盾" |
| **语法审查** | 病句、搭配不当、成分残缺、重复啰嗦 | "'进行了一次全面而深刻的检查'——删去'而深刻的'" |
| **规范审查** | 公文用语、政治表述、格式规范、标点使用 | "该表述不符合最新文件精神，应改用 XXX" |

#### 7.7.3 审查结果展示

审查完成后会展示：
- 每个技能的审查结果：发现了几个问题、修复了几个
- 问题清单：按严重程度（高/中/低）分类
- **DIFF 回放动画**：逐操作高亮展示 AI 建议的改动（红色删除、绿色新增），你可以**逐条确认或拒绝**每个改动

> **技巧**：审查阶段的 DIFF 回放非常有用——它不是简单地让你接受或拒绝全文改动，而是**逐条操作**。你可以接受逻辑修正，但拒绝某条你不认同的语法改动。

### 7.8 阶段五：润色定稿（Polish）—— 详细操作指南

审查完成后，进入润色阶段。不同菜谱使用不同的润色技能：

| 技能 | 效果 | 适用文种 |
|------|------|----------|
| **skill_spirit** | 符合上级精神，提升政治站位 | 述职报告、研讨材料 |
| **skill_golden** | 打造金句，增强感染力和记忆点 | 领导讲话、经验材料 |
| **skill_concise** | 精简凝练，去除冗余 | 工作总结、通讯简报、工作通知、规章制度、课题研究、工作汇报 |

#### 7.8.1 润色操作

1. AI 生成润色版本
2. 展示原文与润色版的 DIFF 对比
3. 你可以：
   - **全部接受**：用润色版替换原文
   - **逐条审阅**：类似审查阶段，逐操作确认/拒绝
   - **重新润色**：不满意可重新生成

### 7.9 流水线的中断与恢复

- 流水线每个阶段的产出**独立保存**
- 任何时候关闭工作台，进度不丢失
- 再次打开同一菜谱会恢复之前的进度
- 你可以在任何阶段暂停，稍后再回来继续

### 7.10 进度条与计时

- 写作工作台启动后，编辑器底部会显示进度条，展示当前阶段和总进度
- 右上角计时器记录总耗时
- 进度条在完成所有阶段后自动消失

## 八、AI 对话

### 8.1 对话面板概述

右栏「AI对话」是与 AI 进行**自由对话**的入口。与写作流水线不同，这里没有固定流程，你可以任意提问。

### 8.2 基本操作

- **发送消息**：在底部输入框中输入问题，回车或点击发送
- **多轮对话**：AI 会记住前后的对话上下文，可以进行连续追问
- **历史消息**：对话界面展示完整的对话记录，支持滚动查看
- **Markdown 渲染**：AI 的回复会以 Markdown 格式渲染展示（表格、列表、代码块等）

### 8.3 对话管理

- **新建对话**：点击「新建对话」创建新的对话会话
- **切换对话**：左侧对话列表展示所有历史对话，点击切换
- **重命名对话**：右键对话标题可以重命名
- **删除对话**：右键对话标题可以删除

### 8.4 引用上下文

你可以将文档内容作为对话的上下文，让 AI 更有针对性地回答：

#### 8.4.1 引用全文

- 开启「**引用全文**」开关
- AI 回复时会以当前编辑器中的全部内容作为参考上下文
- 适合问"全文还有哪里需要修改"、"帮我整体评估一下这篇稿子"之类的问题

#### 8.4.2 引用选区

- 在编辑器中**选中一段文字**
- 右键 →「添加到 AI 对话」
- 选中的文字会以**引用气泡**的形式出现在 AI 对话的输入框上方
- 发送问题时，AI 会以这段文字为上下文参考
- 点击 × 可以取消引用

> **技巧**：针对某一段落的具体问题（如"这段的语气是否合适"、"帮我把这段改得更有力度"），先用选区引用这段文字，再提问，效果远好于全文引用。

### 8.5 引入知识库

- 点击输入框旁边的知识库图标
- 在弹出的知识库选择器中勾选要引用的知识库文档
- AI 回复时会检索这些知识库中的相关内容作为参考

### 8.6 引入素材库

- 在 AI 对话中同样可以选择要引用的素材标签
- 素材标签选择器位于对话面板中
- AI 回复时会检索选中标签下的素材

### 8.7 附件上传

- 点击输入框旁的附件（📎）图标
- 选择本地文件（支持 txt、md）
- 文件内容会作为对话上下文的一部分
- 上传文件以附件气泡展示，可随时移除

## 九、AI 技能与技能组合

### 9.1 什么是 AI 技能

AI 技能是**预设的专项处理能力**——每个技能有特定的 prompt 模板、temperature 参数和输出格式。技能可以：

- **独立调用**：选中技能，对当前文档执行一次处理
- **作为流水线环节**：审查和润色阶段会自动调用对应技能
- **组合成管道**：多个技能按顺序依次执行

### 9.2 内置技能列表

| 技能 ID | 名称 | 分类 | 用途 |
|---------|------|------|------|
| `skill_logic` | 逻辑审查 | 审查 | 检查论点一致性、论证充分性、前后矛盾等逻辑问题 |
| `skill_grammar` | 语法审查 | 审查 | 检查病句、搭配不当、成分残缺、重复等语法问题 |
| `skill_official` | 规范审查 | 审查 | 检查公文用语、政治表述、格式规范和标点使用 |
| `skill_spirit` | 上级精神 | 润色 | 使文稿表述符合上级最新精神和政策导向 |
| `skill_golden` | 金句打磨 | 润色 | 打造金句，增强文稿的感染力和记忆点 |
| `skill_concise` | 精简凝练 | 润色 | 去除冗余表述，使语言更加精炼有力 |
| `skill_rhythm` | 节奏韵律 | 专项 | 分析句式节奏和修辞韵律，适合讲话稿优化 |
| `skill_leader` | 领导用语 | 专项 | 学习领导讲话措辞风格，辅助语气调整 |
| `skill_citation` | 引用检索 | 专项 | 自动检索素材库/知识库内容补充引用 |
| `__compose_generate__` | 正文撰写 | 内部 | 写作流水线正文撰写阶段的专用内部技能 |

### 9.3 独立使用技能

1. 切换到右栏「**技能**」标签
2. 在技能列表中找到需要的技能
3. 可选：勾选要引用为上下文的知识库
4. 可选：勾选要引用的素材标签
5. 点击技能旁的「**运行**」按钮
6. AI 开始处理，处理完成后展示结果
7. 展示原文与结果之间的 DIFF 对比
8. 可以**接受结果**替换原文，或**放弃**保留原文

> **技巧**：写完一段重要内容后，可以分别用"语法审查"和"精简凝练"技能处理一遍。先用语法审查修正错误，再用精简凝练打磨文字。

### 9.4 技能组合管道

这是技能功能的高级用法——**将多个技能串联起来，按顺序依次执行**：

#### 9.4.1 创建管道

1. 在技能面板中，点击「**创建管道**」
2. 在弹出的技能选择器中，按顺序添加技能
3. 技能会按照你添加的顺序依次执行
4. 前一个技能的**输出**会成为下一个技能的**输入**

#### 9.4.2 管道执行

1. 点击「**运行管道**」
2. AI 按顺序执行每个技能
3. 执行过程中显示当前步骤（如"正在执行第 2/3 步：语法审查"）
4. 全部执行完毕后，展示最终结果与原稿的 DIFF 对比

#### 9.4.3 推荐技能组合

| 组合名称 | 技能顺序 | 适用场景 |
|----------|----------|----------|
| **基础打磨** | 语法审查 → 精简凝练 | 写完任意文稿后 |
| **输出定稿** | 逻辑审查 → 语法审查 → 规范审查 → 精简凝练 | 最终定稿前 |
| **讲话稿润色** | 节奏韵律 → 领导用语 → 金句打磨 | 领导讲话稿 |
| **规范达标** | 逻辑审查 → 规范审查 → 上级精神 | 正式呈报材料 |

> **技巧**：可以把常用的技能组合保存下来。比如你经常写述职报告，每次写完都执行"语法→精简→上级精神"三件套，那就创建一个管道并记住组合。每次写完一键运行即可。

### 9.5 自定义技能

除了内置技能，你还可以创建自己的技能：

1. 在技能面板中点击「**新建技能**」
2. 填写以下内容：
   - **技能名称**：如"补充数据支撑"
   - **分类**：自定义或选择现有分类
   - **提示词模板**：这是技能的核心——描述 AI 要做什么、怎么做。支持 `{content}` 占位符表示当前文档内容
   - **Temperature**：设置生成温度（0~1，越低越保守）
3. 保存后，自定义技能出现在技能列表中
4. 可以随时编辑或删除自定义技能

> **技巧**：自定义技能的提示词模板是关键。好的模板应该包含：
> 1. 明确的角色设定（如"你是一位资深编辑"）
> 2. 清晰的任务描述（如"对以下文本进行 X 处理"）
> 3. 具体的输出要求（如"不要改动原文意思，只修改 Y 方面"）

### 9.6 技能中的 Diff 对比

运行技能后，结果会以精确的 DIFF 形式展示：
- **红色标记**：AI 新增的内容
- **删除线标记**：AI 建议删除的内容
- **灰色不变**：原文保留的部分

这使得你可以**非常精确地**看到 AI 到底改了哪些地方，逐字确认。

## 十、知识库管理

知识库是 AiPen 的"外脑"——存放你的参考资料，供 AI 在写作时检索。

### 10.1 知识库概述

知识库与素材库的区别：
- **素材库**：短文本片段（金句、案例、政策摘录），按标签组织
- **知识库**：长篇文档（政策文件全文、法规全文、研究报告），独立管理

### 10.2 创建知识库

**方式一：手动创建**
1. 切换到右栏「知识库」标签
2. 点击「新建」
3. 输入知识库名称
4. 粘贴或输入文档内容
5. 保存

**方式二：文件导入**
1. 点击「导入」按钮
2. 选择本地文件
3. 支持格式：**txt、md、docx、pdf**
4. 文件解析后，自动以文件名作为知识库名称
5. 可在弹出的确认框中修改名称
6. 点击保存

### 10.3 查看与编辑

- 点击知识库条目展开预览内容
- 点击「编辑」可修改名称和内容
- 点击「删除」可删除知识库

### 10.4 知识库如何被 AI 使用

- **写作流水线**：每种菜谱预设了默认关联的知识库（在菜谱卡片上有标注），启动时会自动勾选
- **AI 对话**：可以手动选择要引用的知识库
- **AI 技能**：运行技能时可以勾选知识库作为上下文

> **技巧**：如果你的单位有常用的政策文件（如安全生产管理条例、公文处理办法等），建议把这些文件导入知识库。之后每次写相关文档时，AI 都会自动参考其中的规定，减少政策合规错误。

## 十一、版本管理、比对与回退

### 11.1 版本管理概述

AiPen 的版本管理是"草稿 + 版本"双层模型：
- **草稿**：实时自动保存的最新工作区，可以反复修改
- **版本**：手动提交的不可变快照，用于里程碑记录和回溯

### 11.2 提交版本

写作到重要节点时（如"初稿完成"、"二稿修订"、"领导批注后修改"）：

1. 点击顶栏「**提交版本**」按钮
2. 在弹出的对话框中输入备注信息（如"初稿：完成全部正文"）
3. 点击确认
4. 系统将当前编辑器内容保存为一个不可变版本，并清空草稿

> **技巧**：建立自己的版本提交习惯：
> - 写完初稿 → 提交版本（备注"初稿"）
> - 领导提了修改意见后改完 → 提交版本（备注"按领导意见修改"）
> - 最终定稿 → 提交版本（备注"定稿"）
> - 这样在任何时候都能回溯到之前的任意版本

### 11.3 查看历史版本

1. 切换到右栏「**版本**」标签
2. 所有版本按时间顺序列出
3. 每个版本显示：版本号（v1, v2, v3...）、提交备注、提交时间
4. 点击版本条目可以**在编辑器中查看该版本的内容**（只读）
5. 查看历史版本时，编辑器顶部会显示版本标签（如"v3: 按领导意见修改"）

### 11.4 退出历史查看

- 点击编辑器顶部的「**返回最新**」按钮
- 编辑器恢复到最新草稿内容
- 也可以直接切换到其他文档

### 11.5 版本回退

如果想把某个历史版本恢复为当前工作版本：

1. 在版本列表中，找到目标版本
2. 点击该版本旁的「**回退**」按钮
3. 该版本的内容覆盖当前草稿
4. 可以在此基础继续编辑或提交新版本

> **注意**：回退会覆盖当前草稿。如果当前草稿有未保存的修改，建议先提交一个版本再做回退。

### 11.6 版本比对（Diff）

对比两个版本之间的差异：

1. 在版本列表中，**依次点击两个版本**（先点击旧版，再点击新版）
2. 旧版标记为"←"，新版标记为"→"
3. 切换到右栏「**Diff**」标签
4. 点击「**对比**」按钮
5. 系统展示两个版本之间的差异：
   - **红色**：新版中新增的内容
   - **删除线**：旧版中被删除的内容
   - **灰色**：没有变化的内容
6. 变更类型分类展示：修改（modified）、删除（deleted）、新增（inserted）

### 11.7 版本操作汇总

| 操作 | 位置 | 说明 |
|------|------|------|
| 提交版本 | 顶栏按钮 | 保存为不可变快照 |
| 查看版本 | 版本标签 | 在编辑器中查看历史版本内容 |
| 回退版本 | 版本标签 | 用历史版本覆盖当前草稿 |
| 选择对比 | 版本标签 | 点击两个版本进行选择 |
| 执行对比 | Diff 标签 | 显示两个版本的差异 |
| 重命名版本 | 版本标签 | 修改版本备注 |
| 删除版本 | 版本标签 | 删除某个版本（不可恢复） |

## 十二、AI 分析与文档评分

### 12.1 文档评分

点击顶栏的 **⭐ 评分** 按钮，AI 会对当前文档进行**五维度综合评分**：

| 维度 | 说明 |
|------|------|
| **思想站位** | 政治站位是否准确，是否体现上级精神 |
| **逻辑结构** | 结构是否合理，层次是否清晰 |
| **内容深度** | 分析是否透彻，见解是否独到 |
| **语言表达** | 文字是否流畅，表达是否精准 |
| **格式规范** | 是否符合公文格式规范 |

每个维度给出 0~100 分的评分和简短评语，最后给出综合总分。

- **绿色**（80+）：整体优秀
- **黄色**（60~79）：基本合格，有提升空间
- **红色**（<60）：需较大改进

> **技巧**：完成初稿后先评分一次，了解薄弱环节，重点修改后再评分，看有没有提高。分数提高的过程就是你写作水平提高的过程。

### 12.2 AI 修订分析

在版本 Diff 对比之后，可以要求 AI 分析两个版本之间的修订质量：

1. 先在版本标签中选择两个版本
2. 切换到右栏「**AI分析**」标签
3. 点击「**分析修订**」按钮
4. AI 会从多个维度分析修订：

| 分析维度 | 内容 |
|----------|------|
| **总体评估** | 修订的方向和效果评价 |
| **思想站位分析** | 政治站位是否提升 |
| **逻辑分析** | 逻辑结构的优化点和退步点 |
| **见解深度** | 内容深度的增加和空洞部分 |
| **表达分析** | 语言表达的高亮和改进空间 |
| **修改分解** | 逐类分析修改的类型和示例 |
| **对比总结** | 新旧版本的关键对比点 |
| **修订建议** | 针对当前版本的具体改进建议 |

> **技巧**：这个功能特别适合做"写作复盘"——对比你的初稿和终稿，让 AI 告诉你到底改进了哪些方面，帮助你找到自己的写作短板。

## 十三、导出 Word 与排版设置

### 13.1 快速导出

1. 确保文档编辑完成
2. 点击顶栏「**导出 Word**」按钮
3. 选择保存路径，输入文件名
4. 点击保存，`.docx` 文件生成完成

### 13.2 排版设置

导出前，建议先检查排版设置。点击顶栏「**⚙ 排版**」按钮打开设置面板：

#### 13.2.1 字体设置

可以为 H1、H2、H3、H4 各级标题分别设置字体、字号和是否加粗：

| 级别 | 默认字体 | 默认字号 | 说明 |
|------|----------|----------|------|
| H1 | 方正小标宋简体 | 二号 | 一级标题（如"一、总体情况"） |
| H2 | 方正黑体简体 | 小三 | 二级标题（如"（一）生产经营"） |
| H3 | 方正楷体简体 | 四号 | 三级标题 |
| H4 | 方正仿宋简体 | 四号 | 四级标题 |

可用字体：方正小标宋简体、方正黑体简体、方正楷体简体、方正仿宋简体、宋体、黑体、楷体、仿宋、微软雅黑、Times New Roman

字号范围：一号（26pt）→ 小五（9pt），采用中文号数制。

#### 13.2.2 正文格式

| 设置项 | 说明 | 默认值 |
|--------|------|--------|
| 正文字号 | 正文段落的基础字号 | 三号（16pt） |
| 行距 | 正文段落行间距 | 28pt |
| 页边距 | 上下左右页边距 | 上3.7cm 下3.5cm 左2.8cm 右2.6cm |
| 纸张大小 | 纸张规格 | A4 |

#### 13.2.3 标题风格

支持六型十四式标题风格（与提纲阶段的风格设定对应），确保导出的 Word 中标题风格与写作时设想的一致。

> **注意**：排版设置是**按文档保存**的——每个文档可以有不同的排版设置。切换到不同文档时，排版设置会自动加载对应文档的配置。

### 13.3 导出注意事项

- 导出按 **MS Word 标准**优化，建议在 Word 中打开查看
- 如果在 WPS 中排版偏差，可尝试关闭 WPS 的"标点溢出"、"网格对齐"等高级选项
- 编辑器中的图片会自动嵌入到 Word 文档中
- 字体和字号设置直接影响导出效果，请确保在导出前确认排版设置

## 十四、自定义写作菜谱

### 14.1 为什么需要自定义菜谱

10 种内置菜谱覆盖了常见文种，但你可能会遇到特殊需求：
- 你的单位有特定格式的公文（如"安全环保周报"）
- 你需要一种内置菜谱没有覆盖的文种（如"慰问信"）
- 你想为某个特定上级领导定制写作风格

自定义菜谱可以让你创建专属的写作模板，反复使用。

### 14.2 创建自定义菜谱

1. 切换到右栏「写作」标签
2. 滚动到菜谱列表底部，找到「**+ 新建自定义菜谱**」
3. 输入基本信息：
   - **菜谱名称**：如"安全环保周报"
   - **菜谱描述**：简要描述用途（如"每周安全环保情况汇报"）
   - **文体分类**：输入一个分类名称

4. 点击「创建」后，AI 会**自动分析你的需求**，生成专属的：
   - **systemPrompt**：AI 在该菜谱中的角色定位和任务说明
   - **问题列表**：针对该文种的采集问题

5. 进入确认编辑面板，你可以：
   - **编辑 systemPrompt**：调整 AI 的角色设定和写作要求
   - **编辑问题**：修改、增加、删除、调整问题顺序
   - **设为必答/选答**：每道问题可切换必答/选答
   - **添加选项**：对于选择题型的问题，可添加预设选项（逗号分隔）
   - **重新生成**：如果不满意 AI 生成的结果，可以点击「重新生成」

6. 确认无误后，点击「保存」

### 14.3 自定义菜谱的技术细节

自定义菜谱创建后，其余四个阶段（提纲、撰写、审查、润色）会自动配置：
- **提纲阶段**：使用通用的提纲生成逻辑
- **正文撰写**：使用 `__compose_generate__` 内部技能
- **质量审查**：与内置菜谱相同，执行逻辑→语法→规范三重审查
- **润色定稿**：默认使用精简凝练技能

> **技巧**：创建自定义菜谱时，问题的设计是关键。好的问题列表应该：
> 1. 覆盖该文种的所有关键信息维度
> 2. 每题聚焦一个问题，不要一题多问
> 3. 必答题控制在 3~5 个（核心信息），其余设为选答
> 4. 为有固定答案范围的问题添加选项（如"报告频率：日报/周报/月报/季报"）

### 14.4 管理自定义菜谱

- **编辑**：在菜谱卡片上点击编辑按钮，修改名称和描述
- **删除**：在菜谱卡片上点击删除按钮
- **导入/导出**：自定义菜谱包含在数据导出中，可随备份一起迁移

## 二十、最佳实践

### 20.1 写作工作流建议

```
日常素材积累 → 知识库建设 → 启动菜谱 → 认真问答 → 确认提纲 → AI生成 →
人工修改 → 三重审查 → 润色定稿 → 评分 → 导出 → 提交版本
```

### 20.2 各阶段使用技巧

#### 素材积累阶段

1. **随手存**：看到好的政策原文、领导讲话、金句名言，立刻存入素材库
2. **勤打标签**：分类越细，检索越准（不要只设"领导讲话"，细分为"领导讲话-安全""领导讲话-党建"）
3. **记录来源**：用浏览器剪藏功能，自动记录来源网址，方便追溯引用

#### 写作准备阶段

4. **先审题后开机**：打开 AiPen 之前，先想清楚：写给谁看、什么场合、什么目的
5. **建好知识库**：把常用的政策文件、上级文件导入知识库，AI 会自动参考
6. **选对菜谱**：如果 10 种内置菜谱不完全匹配，创建自定义菜谱

#### 问答采集阶段

7. **认真回答必答题**：必答题是 AI 理解需求的关键，跳过了 AI 只能靠猜
8. **选答题巧用跳过**：不确定的选答题直接跳过，比瞎编一个答案好
9. **善用提示词**：把常用的答案模板保存为提示词，下次一键插入
10. **摘要窗口不要跳过**：这是调整方向的最佳时机，花 2 分钟仔细过一遍

#### 提纲阶段

11. **先确认结构**：结构决定文章的骨架，结构有问题正文怎么写都不对
12. **风格不是装饰**：选对标题风格能让全文格调提升一个档次
13. **大胆修改**：提纲是 AI 的建议，不是圣旨，任何不满意的条目都可以修改

#### 正文撰写阶段

14. **不要打断**：AI 在逐节生成时，让它写完一节再看，频繁暂停打断上下文
15. **节节确认**：每节生成完快速扫一眼，发现问题立刻让 AI 重新生成那节
16. **保留原文**：如果某节 AI 写得不够好但你觉得方向对，先留着，到审查阶段再修

#### 审查润色阶段

17. **逐条看 DIFF**：审查的回放动画要逐条看，不要一键全部接受
18. **逻辑优先**：先确认逻辑没有问题，再管语法和修辞
19. **善用技能管道**：做完智能审查后，再用技能管道跑一遍自定义的组合

#### 定稿阶段

20. **先评分再导出**：导出前评分一次，看看有没有漏掉的问题
21. **提交定稿版本**：导出 Word 后，也记得在 AiPen 中提交一个"定稿"版本
22. **复盘修订**：用 AI 分析功能对比初稿和定稿，看看自己进步在哪里

### 20.3 效率提升要点

| 技巧 | 效果 |
|------|------|
| 页面缩放 80% 审阅长文 | 一屏看到更多内容，适合对比审阅 |
| 页面缩放 120% 精修 | 字大眼不累，适合逐字打磨 |
| 大纲始终打开 | 随时把握全局结构 |
| 主题跟随系统 | 白天浅色、晚上深色，自动切换护眼 |

## 二十一、常见问题

### Q1：API 配置后测试连接失败？

A：请依次检查：
1. API 密钥是否正确复制（确保没有多余空格，以 `sk-` 开头）
2. 账户是否已完成**实名认证**（未认证无法调用 API）
3. 账户**余额是否充足**（余额为 0 时调用失败）
4. 网络是否正常（可以访问 `https://api.deepseek.com`）
5. 点击设置中的「测试连接」查看具体错误信息

### Q2：AI 调用很慢怎么办？

A：可能的原因和解决方法：
- 检查是否开启了「思考模式 max」——尝试降为 high 或关闭思考模式
- 检查当前使用的模型——V4 Pro 比 Flash 慢很多，日常使用建议 Flash
- 检查网络状况——如果使用代理，确保代理稳定
- 如果是长文生成，分段生成比一次性生成全文更快

### Q3：启动 / 窗口切换时短暂闪屏？

A：已通过三道防线优化：
- Tauri 窗口背景预设为中性色，消除 OS 层默认白板
- HTML 首帧即正确主题色，不等外部 CSS
- 同步读取主题设置，提前标记主题 class

深浅模式通用，启动切换几乎不可感知。

### Q4：保存失败、提示红色错误？

A：请检查：
- 磁盘剩余空间是否充足
- 文件是否被其他程序锁定（如 Word 打开了同名 .docx）
- 应用是否有写入权限（某些企业安全软件可能拦截）

### Q5：导出的 Word 在 WPS 中排版错位？

A：目前导出按 **MS Word 标准**优化排版，建议在 Word 中打开查看。如果必须在 WPS 中编辑：
- 在 WPS 中关闭「标点溢出」选项
- 关闭「网格对齐」选项
- 或者导出后在 Word 中打开，再另存为兼容格式

### Q6：如何在不同电脑上同步数据？

A：使用数据导出/导入功能：
1. 在旧电脑上，进入设置 → 导出选中数据
2. 将 `.aipen` 文件传输到新电脑（U 盘、云盘等）
3. 在新电脑上，进入设置 → 导入数据
4. 重新配置 API 密钥（备份文件不包含密钥）

### Q7：软件可以离线使用吗？

A：基本编辑功能（打字、排版、素材管理）可以离线使用。所有 AI 功能（写作流水线、对话、技能、评分、分析）需要网络连接调用 DeepSeek API。

### Q8：如何重置应用全部数据？

A：删除 `%APPDATA%\AiPen` 目录即可。**操作前务必先做完整备份**。
