## v2.0.6 (2026-07-04)

> 本次更新聚焦于**编辑器交互体验**:工具栏重排、深色模式质感升级、防白闪优化。

### ✨ 优化与改进

#### 编辑器工具栏重排
- **大纲导航置顶**:大纲按钮移至工具栏最左侧,作为首要导航入口
- **按功能属性分组**:将原 14+ 个按钮按"导航 / 编辑历史 / 基础文字格式 / 高级文字样式 / 段落样式 / 段落排列 / 列表 / 插入元素 / 文档工具 / 字体字号"分组,组间用分隔符清晰分割
- **合并冗余分组**:原"缩进"组与"对齐"组合并为"段落排列",原"查找替换"与"打印"合并为"文档工具",语义更清晰
- 整体布局参考主流文字处理器,**符合用户认知习惯**

#### 深色模式质感全面升级
解决此前深色模式"发闷、不通透、字体发暗"的问题,采用**三层深度体系**重构配色:

| 层级 | 区域 | 旧色 | 新色 |
|------|------|------|------|
| 最深(暗) | 工具栏 | `#1e1e2e` | `#14151d` |
| 中间 | 下拉面板 | `#252540` | `#1a1c29` |
| 最亮(画布) | 编辑器内容区 | `#1a1b26` | `#1d2032` |

- **文字提亮**:`#c0caf5` → `#cdd6f4`,久写不累
- **激活态精致化**:`#60a5fa` → `#89b4fa`,更现代通透
- **阴影升级**:下拉面板 16px → 28px,右键菜单 24px → 36px,营造玻璃悬浮感
- **元信息可见度提升**:`rgba(255,255,255,0.04)` → `0.06`
- 占位符、引用块、代码块、搜索高亮、上下文危险悬停等 11 项 CSS 变量同步调整
- 浅色模式保持不变,无回归

#### 防白闪三道防线
解决 Tauri 应用启动 / 窗口切换时的**白屏闪烁**问题,深浅模式通用。

| 防线 | 防御对象 | 实现 |
|------|----------|------|
| **第一道** | OS 窗口 client area 默认白 | `tauri.conf.json` `backgroundColor: #2a2c3c`(中性色) |
| **第二道** | 浏览器 body 默认白(CSS 加载竞速) | `index.html` 内联 CSS 变量,首帧即正确主题色 |
| **第三道** | 主题 class 注入延迟(JS 加载竞速) | 同步 IIFE 提前打 `.dark`,纳秒级 |

- **中性色 `#2a2c3c`** 介于浅色和深色之间,深色用户从 ΔL=88 的"白→深"突变降为 ΔL=30 的"中性→深",感知不到
- 浅色用户从"白→白"无感,不会被反向突兀

#### 主题系统重构(user/system 双模式)
- 新增 `source` 字段,区分"用户显式选择"与"跟随系统主题"
- 首次启动默认**跟随系统主题**(`prefers-color-scheme`)
- 系统主题切换时自动跟随(仅在 source=system 时响应)
- 用户手动切换后 source=user,持久化到 localStorage
- 切回"跟随系统"会清掉显式选择记录,避免污染
- `useTheme()` API **完全向后兼容**,新增 `followSystem()` 方法

### 🐛 修复

- 浅色 / 深色模式切换时偶发的"白闪"问题(三道防线彻底根除)
- 启动后首帧文字颜色短暂为深色的问题(同步 IIFE 在 CSS 应用前完成 class 标记)

### 📝 兼容性说明

- v2.0.5 及更早版本的 `localStorage` 中只有 `aipen-theme` 键,无 `aipen-theme-source` 键
- 老用户首次启动 v2.0.6 时,会被识别为"无 source 记录",**默认转为跟随系统**
- 如希望保留旧行为(总是显式选择),启动后**手动切换一次**主题即可固化

## v2.0.5 (2026-07-04)

### 🏗️ 架构升级

#### 存储格式全面切换为 ProseMirror JSON
- **编辑器引擎**:从 Milkdown(Markdown)迁移至 **TipTap (ProseMirror)**,实现所见即所得的富文本编辑
- **存储格式**:文档内容以 ProseMirror JSON 格式存储(`{type:"doc", content:[...]}`),保留完整排版语义
- **向后兼容**:旧 Markdown 格式数据启动时自动通过 `textToDocJson` 转换为新格式
- **导出不变**:导出 Word 仍按公文标准排版,XML 后处理完整保留

#### AI 调用纯文本提纯优化
- 新增 `prose_mirror_to_plain_text()` Rust 函数,在所有 AI 调用入口处自动将 ProseMirror JSON 提取为纯文本
- 覆盖范围:Diff 对比、AI 分析、文档评分、AI 对话、技能执行
- **效果**:节省 2~3 倍 token 消耗,20,000 字文档从 ~8000 token 降至 ~2800 token

### ✨ 新增功能

#### 素材库 & 浏览器剪藏
- 左侧面板新增「素材」子标签页,管理写作素材
- **标签系统**:创建/重命名/删除标签,按标签筛选素材
- **存入素材库**:编辑器右键选中文字 → 存入素材库,AI 自动分析并建议标签
- **素材编辑**:点击素材进入编辑模式,内容自动保存
- **标签文档**:按标签聚合全部素材为只读文档,便于统一浏览
- **浏览器剪藏**:内嵌 WebView 浏览器,支持书签管理、网页浏览、右键选中文字存入素材库
- **双通道剪藏**:自定义 URI scheme + eval 双通道保障,防止 WebView2 CSP 限制

#### 文档综合评分
- 标题栏旁 ⭐ 评分按钮,AI 对当前文档进行 0~100 分综合评分
- 五维度评析(内容质量、逻辑结构、语言表达、规范合规等),带进度条和评语
- 评分结果自动缓存,切换文档直接显示
- 支持历史版本独立评分

#### 动态保存状态
- 状态栏实时显示保存流转:`检测到变更…`(黄色)→ `正在保存…`(蓝色)→ `草稿已自动保存(23:14:05)`(绿色)
- 保存失败时显示红色错误提示
- 时间精确到秒

#### 智能写作流水线增强
- **菜谱系统**:10 种内置菜谱 + 自定义菜谱,AI 自动生成采访问题
- **六型十四式标题风格库**:结构强化/逻辑关系/意象赋能/词汇创新/价值引领/数字统领
- **DIFF 回放动画**:审查/润色完成后,逐操作高亮动画展示每一处改动
- **流式推送**:正文生成、审查、润色全部流式推送到编辑器

#### 查找替换
- Ctrl+F 打开查找替换面板,支持大小写、正则表达式、全词匹配
- 逐个替换 + 全部替换,界面完整汉化

### 🔧 改进与优化

#### 编辑器
- 10 种中文字体可选(等线、微软雅黑、宋体、黑体、楷体、仿宋等)
- 主题切换(深色/浅色),偏好自动保存
- 标题/段落折叠,行号高亮
- Ctrl+滚轮 缩放字体 10~32px
- 右键菜单集成「存入素材库」「添加到 AI 对话」

#### 文档管理
- 文件夹完整管理流程(创建/重命名/删除/筛选/移动文档)
- 切换 Tab 自动保存
- 文档列表显示更新时间

#### 右侧面板
- 重新组织为 8 个标签页:版本 / Diff / AI分析 / AI对话 / 技能 / 写作 / 知识库 / 设置
- 侧栏折叠按钮,最大化编辑空间

#### AI 对话
- 右键引用全文 + 右键引用选中文本
- 支持知识库和素材库作为对话上下文
- 选中文本后自动切换到对话页

#### 导出 Word
- 完整的 XML 后处理:禁用孤行控制、标点溢出、网格对齐、表格内缩进归零
- 页脚居中页码
- 支持 WebP/SVG 图片自动转 PNG

#### 数据管理
- 按类别勾选导出/导入 .aipen 备份文件
- 导入自动去重(同名文档合并版本、同名文件夹跳过)
- 支持数据统计面板

### 🐛 修复

- **Diff 死循环**:修复 ProseMirror JSON 直接做逐字符 diff 导致前端 changeBlocks 假死的问题,现在先提取纯文本再做行级 diff
- **AI 分析 token 浪费**:`analyze_revision` 和 `score_document` 现在自动提取纯文本后发给 AI
- **版本对比格式**:diff 输出现在是用户可读的文本差异,而非 JSON 结构差异
- **保存状态**:从永远显示「草稿已自动保存」改为动态流转状态

### 📝 已知限制

- `documentPlainText`(前端纯文本提取)不保留段落换行,后端提取保留段落结构——两者在文本格式上存在细微差异
